# BASE管理画面 構造調査結果
調査日：2026-07-12
※個人情報(氏名・住所・連絡先等)は本書に一切含めない。構造・項目名・形式のみ記録。

---

## 1. 重要な発見：ステータスと業務実態の乖離

注文詳細に管理者自身のメモとして以下の運用実態が記録されていた：

> 「便宜的にステータスを発送済に変更してありますが、商品はまだ発送されていません」

**分析**：BASEの注文ステータス(入金待ち/未対応/対応中/対応済/キャンセル)は数日で発送する物販前提の設計であり、16〜18週間の受注生産には適合しない。BASEの発送期限制約を回避するため、実態と異なるステータス操作を強いられている。**受注〜納品の一元管理システムが必要な根本理由はここにある。**

→ 新システムでは「決済ステータス」と「生産・納品ステータス」を完全に分離して設計する。

## 2. 注文データ構造

### 注文ID
16桁の英数字(例形式：`04B6D9741F7DDE65`)

### 注文ステータス(BASE)
入金待ち / 未対応 / 対応中 / 対応済 / キャンセル

### 注文詳細の保持項目
| 区分 | 項目 |
|------|------|
| 注文情報 | 注文ID / 注文日時 / 購入元(Webショップ PC等) / 決済方法 / 合計金額 / 商品小計 / 送料 |
| 購入商品 | 商品名 / 種類(=ペン機種バリエーション) / 単価 / 配送方法 / オプション(複数、各に価格差分) / 購入数 |
| 配送情報 | 発送日時 / 配送会社 / 伝票番号 |
| 顧客入力 | 購入者へのメッセージ / 購入者からの要望 / 配送希望日時 |
| お届け先 | 氏名 / 郵便番号 / 住所 / 電話番号 |
| 購入者 | 氏名・住所・電話(お届け先と同一可) / メールアドレス |
| 顧客集計 | 購入回数 / キャンセル回数 |
| 管理用 | 注文メモ |

### 注文明細の実例構造(値は形式のみ)
```
商品：グレイングリップ LITE ブラウン
種類：【Wacom】Pro Pen3(軸の中におもり)ACP50000DZ
オプション：
  【グリップの形状】：【Propen3用】純正のストレート軸と同じ形状(¥0)
  表面の仕上げ：マット仕上げ(天然塗料を塗布)(¥0)
  ボタンの調整：ボタン穴あり 純正と同じ押し具合(¥0)
```

## 4. 商品オプション構造(全10グループ)

BASEではオプショングループをショップ共通で定義し、商品ごとに割当てる方式。

| グループ | 選択肢数 | 内容 |
|---------|---------|------|
| 【グリップの形状】 | 6 | 標準径 / 太径 / 倒立タイプ / 細径 / Propen3用×2 |
| コルクグリップ 形状 | 3 | 標準径 / 太径 / 倒立タイプ |
| Apple Pencil 機種 | 3 | 第一世代 / 第二世代 / Apple Pencil PRO |
| 製品仕様(Apple Pencil) | 14 | つけペン/ペンタブ×タッチ可否、太径×充電/タッチ、三角〜八角軸、特注形状 |
| 利き手 | 2 | 右利き / 左利き |
| インク染め | 24 | PILOT色彩雫20色+他社インク+特注指定 ※ニス仕上げ推奨の注記付き |
| グリップの形状 | 14 | 標準/細/太軸、エルゴノミクス(フルオーダー)、三角〜八角軸、ProPen3用5形状 |
| 表面の仕上げ | 4 | マット(天然塗料) / 蜜蝋 / 耐水ポリウレタンニス / 滑り止めニス(光沢) |
| ボタンの調整 | 5 | 穴あり / 穴なし / 先端側誤動作防止 / 後端側誤動作防止 / 両方 |
| 【ロゴの焼印の有無】 | 1〜 | ロゴなし(他選択肢は要確認) |

**設計への示唆**：
- オプションは「グループマスタ+選択肢マスタ(価格差分付き)+商品への割当」の3層構造が必要
- 「インク染め」「利き手」「フルオーダーメイド」など要件定義書v1.1に未記載の重要オプションが存在
- 同名に近いグループが複数ある(【グリップの形状】とグリップの形状)→新システムでは整理統合の余地

## 6. 顧客データ

- 顧客数：**約1,300件**(52ページ×25件/ページ)
- 保持項目：氏名 / 顧客メモ / 購入回数 / キャンセル回数 / 最終購入日
- 顧客グループ機能あり
- BASEの顧客モデルは薄く、実質的な顧客データは注文レコード側に存在する
- キャンセル歴のある顧客が識別可能

**移行への示唆**：顧客マスタは注文CSVから逆生成する方が完全(メール・住所は注文側にしかない)。

---

## 7. 新システム設計への主要な示唆

1. **決済ステータスと生産ステータスの分離**が最重要(BASEの根本的欠陥の解消)
2. 商品構造は「商品 → バリエーション(ペン機種) → オプション(10グループ)」の3階層
3. 受注受付制御は現在バリエーション単位の在庫数で代用→新システムでは明示的な受注開閉フラグ+キュー連動自動制御へ
4. 国内/海外の二重価格体系をどう統合するか要検討(送料別建てにすれば単一価格にできる)
5. 顧客約1,300件・注文履歴の移行はCSVエクスポートから実施
6. フルオーダーメイド(エルゴノミクス形状)という受注区分が存在→通常受注と別フローの可能性
