# 旧販売サイト閉鎖チェックリスト(TASK-34)

実施タイミング:TASK-33完了後、新サイトが安定稼働してから2週間以降を推奨(指示書の依存関係のとおり)。

対象:英語BASE(pinion.thebase.in)・Booth・Stores(日本語BASEはTASK-33でリダイレクト対応済み)。

## 各プラットフォーム共通の手順

- [ ] `docs/closure-announcements.md`の告知文(該当言語)を、トップお知らせ+商品説明冒頭に掲示
- [ ] 掲示開始日を記録(2週間後が受注停止予定日の目安)
- [ ] 2週間経過後、新規注文の受付を停止(プラットフォームの設定で「販売停止」等の操作)
- [ ] 受付停止後、`supabase/verification/pre_closure_check.sql`を実行し、そのプラットフォームの
      最新CSVエクスポート分が全て新システムに取込済みであることを確認(未取込があれば個別に
      BASEインポート機能(A-16)で取り込んでから次へ進む)
- [ ] 退店・解約前に、そのプラットフォームの全データ(注文・顧客・商品)の最終CSVエクスポートを
      取得し、ローカル+クラウドの二重保管を行う(保管場所は`docs/migration_complete.md`に記録)
- [ ] プラットフォームの解約・退店手続きを実施
- [ ] `docs/migration_complete.md`の該当行(告知開始日/受注停止日/閉鎖日)を更新

## プラットフォーム別の個別確認事項

### 英語BASE(pinion.thebase.in)
- [ ] 独自ドメインを設定していないか確認(していれば退店前にDNS設定を解除)
- [ ] 海外顧客向けの告知のため、英語版の告知文を使用したことを確認

### Booth
- [ ] 出品中の全商品ページに閉鎖告知(短縮版)を追記したか確認
- [ ] Boothの退店手続きページの案内に従う(サービス仕様は変わり得るため、実施時に最新の
      退店手順を確認すること)

### Stores
- [ ] 出品中の全商品ページに閉鎖告知(短縮版)を追記したか確認
- [ ] Storesの解約手続きページの案内に従う(同上、実施時に最新手順を確認)

## 顧客への移行案内メール(任意)
